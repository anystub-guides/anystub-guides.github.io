Zero efforts stubbing for your integration tests in Java

Why do you need it

  • make your app independent from external data sources
  • test classes, subsystems or the whole app against real data
  • zero efforts to collect test data
  • make your tests autonomous and running at any isolated build server

anyStub provides simple API and requires minimum code overhead to get benefits.

How to cook it

Out of box anyStub supports stubbing http and jdbc connections. To stub your connection you need to wrap your connection with a wrapper and let it work. For stubbin http connection use org.anystub.http.StubHttpClient, for jdbc dataSources org.anystub.jdbc.StubDataSource. In real project you reraly create the objects. More likely you have to create/configure a builder or factory which will create the connections or even spring-boot will create everyting for you. Examples below demonstrate how to stub real connections in the cases.

Once you stub the connections all external calls will appear in a default-stub file src/test/resources/anystub/stub.yml Sometimes you need to split the calls in different stub-files. You can utilize @AnystubId annotation. It's a class/method level annotation. You can put it with your test class/test method to route all calls under the annotation in a distinct file. By default the file has a name of TestClassName.yml/TestMethodName.yml and puts in src/test/resources/anystub/. The annotation has a parameter "filename" to give the file specific name. The extension '.yml' will be added automatically
